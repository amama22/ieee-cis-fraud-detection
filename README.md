# IEEE-CIS Fraud Detection - მანქანური სწავლების ექსპერიმენტები

## Kaggle კონკურსის მიმოხილვა

IEEE-CIS Fraud Detection კონკურსი Kaggle-ზე მოიცავს საბანკო 
ტრანზაქციების ანალიზს თაღლითური ოპერაციების აღმოსაჩენად. 
მონაცემთა ბაზა შედგება ორი ცხრილისაგან — transaction და identity. 
identity ცხრილი მხოლოდ ტრანზაქციების 24.4%-ს მოიცავს, 
დანარჩენებს identity ინფორმაცია არ გააჩნიათ. სამიზნე 
ცვლადი არის isFraud (ბინარული), შეფასების მეტრიკა — ROC-AUC.
კლასების დისბალანსი მნიშვნელოვანია: ტრანზაქციების მხოლოდ 
3.5% არის თაღლითური.

## Feature Engineering

### კატეგორიული ცვლადების რიცხვითში გადაყვანა

სამი განსხვავებული მიდგომა გამოვიყენეთ სხვადასხვა 
მოდელსა და ვარიაციაში.

პირველი მიდგომა იყო OrdinalEncoder, რომელიც გამოვიყენეთ 
XGBoost-ის ყველა ვარიაციაში, LightGBM V1-ში და RandomForest-ის 
ყველა ვარიაციაში. handle_unknown პარამეტრი დავაყენეთ 
use_encoded_value-ზე, რათა test set-ში უცნობი კატეგორიები 
-1-ად გადაქცეულიყო და pipeline არ გაფუჭებულიყო.

მეორე მიდგომა იყო LightGBM-ის native category dtype, 
რომელიც გამოვიყენეთ LightGBM V2 და V3-ში. სვეტები 
რჩებოდა category dtype-ად და LightGBM მათ შიდა 
ოპტიმიზაციით ამუშავებდა.

მესამე მიდგომა იყო Frequency Encoding, რომელიც 
გამოვიყენეთ P_emaildomain სვეტისთვის LightGBM და 
XGBoost-ის ზოგიერთ ვარიაციაში. სვეტი შეიცვალა 
train set-ში ამ დომენის გამოჩენის სიხშირით, რაც 
ინფორმაციის დაკარგვის გარეშე ინახავს დომენის 
პოპულარობის სიგნალს.

M სვეტები (M1-M9) შეიცავდა T/F/NaN მნიშვნელობებს. 
NaN გადავაქციეთ -1-ად და არა 0-ად, რადგან შეუმოწმებელი
განსხვავდება არ დამთხვევისგან. M1 სვეტი 
ამოვიღეთ იქნა ყველა ვარიანტში, რადგან 319,415 T 
მნიშვნელობის წინააღმდეგ მხოლოდ 25 F იყო, 
პრაქტიკულად ნულოვანი ვარიაცია.

### NaN მნიშვნელობების დამუშავება

XGBoost და LightGBM ამუშავებენ NaN-ს native-ად, 
ამიტომ numeric imputation საჭირო არ იყო. LightGBM-ში 
V2 და V3 ვარიაციებში კატეგორიული სვეტები category 
dtype-ად დარჩა, რაც LightGBM-ს საშუალებას აძლევდა 
NaN-ებიც ოპტიმალურად დაემუშავებინა.

RandomForest-ი ვერ ამუშავებს NaN-ს, ამიტომ imputation 
სავალდებულო იყო. V1-ში გამოვიყენეთ median imputation, 
რომელიც outlier-ებისადმი გამძლეა. V2-ში გამოვიყენეთ 
mean imputation, უფრო მარტივი baseline-ისთვის. V3-ში 
გამოვიყენეთ median imputation + missingness indicator 
flags: imputation-მდე დავამატეთ 
binary სვეტი თითოეული სვეტისთვის, რომელსაც >20% 
null ჰქონდა. ამ მიდგომამ 421 სვეტიდან 663-მდე 
გაზარდა feature სივრცე, სანამ RF importance selection 
150-მდე დაიყვანდა.

ასახულიყო.

D1_is_new_account შეიქმნა LightGBM V2-V3 და RandomForest

## Cleaning მიდგომები

null სვეტების წასაშლელად სამი განსხვავებული ზღვარი 
გამოვიყენეთ. V1-ში ვშლიდით >80% null სვეტებს, 
რაც ბალანსს ინარჩუნებდა ინფორმაციასა და ხმაურს 
შორის და დაახლოებით 357 სვეტს ტოვებდა. V2-ში 
ვშლიდით >50% null სვეტებს, რაც აგრესიული მიდგომა 
იყო და დაახლოებით 217 სვეტს ტოვებდა, მაგრამ 
identity სვეტების უმეტესობასაც შლიდა (მათ შორის 
id_01-ს, რომელიც has_identity feature-ის საფუძველი 
იყო). V3-ში ვშლიდით მხოლოდ >90% null სვეტებს, 
რაც კონსერვატიული მიდგომა იყო და 420+ სვეტს 
ტოვებდა, მაღალი noise-ის რისკით.

LightGBM-ის ნოუთბუქში D4 და D15 სვეტებში 
აღმოჩნდა უარყოფითი მნიშვნელობები (D4 min=-122). 
ეს მნიშვნელობები განულდა რადგან დღეების 
რაოდენობა უარყოფითი ვერ იქნება.

## Feature Selection

სამი განსხვავებული მიდგომა გამოვიყენეთ.

Variance Threshold გამოვიყენეთ V1-ში ყველა მოდელში. 
ამ მიდგომით ვშლიდით სვეტებს, რომელთა ვარიაცია 0.01-ზე 
ნაკლები იყო. ამოღებულ სვეტებში აღმოჩნდა M4 და 
რამდენიმე V სვეტი. ეს მეთოდი სწრაფია, მაგრამ 
ვერ ითვალისწინებს სვეტებს შორის კავშირებს.

Correlation Filter გამოვიყენეთ V2-ში ყველა მოდელში. 
ვშლიდით სვეტებს, რომლებიც 0.95-ზე მაღალ კორელაციას 
ავლენდნენ სხვა სვეტებთან. V სვეტების family-ები 
ძლიერ კორელაციაშია ერთმანეთთან, ამიტომ 65-მდე 
სვეტი ამოვარდა. ეს მეთოდი გვარიდებს
redundant feature-ებს, თუმცა შეიძლება ინფორმაციული 
სვეტებიც შეიწიროს.

Feature Importance based Selection გამოვიყენეთ V3-ში 
ყველა მოდელში. 50,000 სტრიქონის sample-ზე სწრაფი 
მოდელის გაშვებით ვთვლიდით feature importance-ებს 
და ვინარჩუნებდით top 150 სვეტს. ეს მეთოდი ყველაზე 
ინფორმაციულია, მაგრამ sample-ზე დაფუძნება შეიძლება 
სრულ dataset-ს ზუსტად არ ასახავდეს.


## Training

### ტესტირებული მოდელები

სამი მოდელი გავტესტეთ: XGBoost, LightGBM და 
RandomForest. თითოეულისთვის გამოვიყენეთ 
StratifiedKFold 3 fold-ით, რაც კლასების 
დისბალანსის სწორ პროპორციას ინარჩუნებდა
ყოველ fold-ში. კლასების დისბალანსი
დავაბალანსეთ scale_pos_weight პარამეტრით 
XGBoost-სა და LightGBM-ში
და class_weight='balanced' პარამეტრით RandomForest-ში.

LightGBM-ი გამოვლინდა საუკეთესო მოდელად tuned 
კონფიგურაციაში, შემდეგ მოდიოდა XGBoost, ბოლოს 
კი RandomForest. RandomForest-ი მთლიანად ჩამორჩა 
gradient boosting მოდელებს, რაც ლოგიკურია, რადგან
gradient boosting კეთ მუშაობს 
tabular მონაცემებზე კლასების დისბალანსის 
პირობებში.

### Hyperparameter ოპტიმიზაციის მიდგომა

თითოეული მოდელისთვის გავტესტეთ 4 კონფიგურაცია, 
რომლებიც განზრახ მოიცავდნენ underfit, baseline, 
overfit და tuned სცენარებს.

Underfit კონფიგურაცია შეიქმნა ძალიან მარტივი 
მოდელით: XGBoost-ში max_depth=2 და n_estimators=50, 
LightGBM-ში num_leaves=8 და min_child_samples=100, 
RandomForest-ში max_depth=5 და min_samples_leaf=100. 
ამ კონფიგურაციებმა აჩვენეს დაბალი val AUC და 
ძალიან მცირე სხვაობა train და val AUC-ს შორის,
underfitting-ის კლასიკური სიმპტომი. 
მოდელი ზედმეტად მარტივი იყო 432-სვეტიანი 
dataset-ის კომპლექსური პატერნების ასათვისებლად.

Overfit კონფიგურაცია შეიქმნა გადამეტებული 
სირთულის მოდელით: XGBoost-ში max_depth=8 
და reg_alpha=reg_lambda=0, LightGBM-ში 
num_leaves=256 და min_child_samples=1, 
RandomForest-ში max_depth=30 და min_samples_leaf=1. 
LightGBM-ის overfit კონფიგურაციამ მიაღწია 
train AUC=1.0000-ს, რაც სრული დამახსოვრების 
ნიშანია. განსაკუთრებით საინტერესო იყო ის, 
რომ num_leaves=256 + min_child_samples=1 
კომბინაციამ გამოიწვია სასწავლო მონაცემების 
სრული დამახსოვრება, თითოეული leaf ფაქტობრივად 
ინდივიდუალური sample-ების შესაბამისი გახდა.

Tuned კონფიგურაციაში შევეცადეთ ბალანსის 
პოვნა. XGBoost-ში გამოვიყენეთ subsample=0.8 
და colsample_bytree=0.8 (ეს პარამეტრები 
ამცირებენ overfitting-ს შემთხვევითობის 
შეტანით ხის აგებისას), reg_alpha=0.1 და 
reg_lambda=1.0 L1/L2 რეგულარიზაციისთვის. 
LightGBM-ში num_leaves=64 (overfit-ის 256-თან 
შედარებით), min_child_samples=20 (overfit-ის 
1-თან შედარებით). RandomForest-ში 
min_samples_split=50 და min_samples_leaf=20
